# Project: *“insert project name”*

* ## Project goal

  * What do we want to accomplish with this project?  
  * In case of academic settings, what is the grade everyone is aiming to get?

* ## Competencies

  * What are everybody's strengths?  
  * What are everybody’s weak points?  
  * Are there any competencies in which anyone would like to improve?

* ## Role division

  * Besides the core workload of any project, there is also workload that is inherently present due to the fact that you’ll be working in a team. E.g. division of labour, and communication. To manage this added workload, which is often overlooked, it is useful to discuss what role everyone will be taking on when working in a team. Examples of roles are: Team Lead, Chief communication, Researcher and Editor.  
  * Discuss which team member will be taking on which role.

* ## Division of Labour

  * Discuss what labour needs to be done for the project.  
  * Divide the labour among all the group members.  
  * Make a timeline of which task will be done when (At least for those tasks that would need to be done before the next meeting).

* ## Concluding the Meeting

  * Determine the topic of the next meeting.  
  * Set a time for the next meeting.  
  * Discuss what tasks will be done before the next meeting.  
  * Discuss if any team members are meeting up among themselves/collaborating with each other before the next meeting. They can plan their meetups after this meeting is concluded.  
  * Close the meeting\! Go have a drink together as teambuilding exercise or something.
